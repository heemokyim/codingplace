만약 어떤 회사에서 해킹탐지시스템을 만들었다 치자
이 해킹시스템은 0시 ~ 24시까지 홈페이지에 머문 분만큼 feature로 기록을 한다.

ex)		0  1 2 ..23 24
User 1  20 0 0   42 52

* 그럼 해킹감지는 구체적으로 어떻게 하나?
1. 해당 유저의 activity vector를 주기적으로 저장한다
2. 이 vector의 평균을 내던 합을 내던 어떻게든 pattern vector를 만든다
3. 의심가는 activity log랑 pattern vector를 내적해서 미리 정해둔 threshold보다 값이 작으면 해킹이라고 판단
반대면 해킹이 아니라고 판단 (유사도 기준(유클리드, 코사인)을 세워서 유사도를 측정)
