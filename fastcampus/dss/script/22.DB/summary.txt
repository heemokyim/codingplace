MySQL 사용 시 생성되는 파일 3가지
= MYD, MYI, FRM

1. MYD = 실제 튜플
2. MYI = 테이블 인덱스에 대한 정보 (B+ tree로 검색해서 SELECT 속도가 빠름)
3. FRM = 테이블 스키마

만약 인덱스가 설정되있다 = MYI에서 트리검색
인덱스가 설정 안되있다 = 테이블 FULL SCAN

** 그럼 어떤 feature를 인덱스로 설정해야 할까?
= 가장 고유한 값을 가지는 컬럼만 (feature끼리 상관관계가 있는 애들은 해줘도 성능향상 별로 안 일어남)
ex)이름, 주소, 우편번호가 있다면 이 중에서 가장 튜플을 잘 나타내는 이름을 인덱스로 설정

** 어떤 놈을 해야 할까보다 어떤 놈을 안해야 할까가 중요한 경우도 있다

** 그럼 인덱스는 언제 자주 쓰이나?
= 주로 SELECT 성능을 향상시킬 때 사용함
= 그 중에서도 JOIN시킬 때
= 따라서 JOIN에서 해외키로 걸리는 컬럼을 인덱스로 걸어주면 좋다

** 인덱스의 맹점
= SELECT는 성능이 향상되나 INSERT, UPDATE, DELETE는 하락함
= 다만 모든 경우에서 이런 게 아니라 데이터 분포에 따라 달라질 수 있음

** 인덱스가 어떻게 성능을 향상시키나
1. SELECT시 DB가 buffer에 데이터가 있는지 확인함
2. 없으면 디스크에서 불러옴
3. 이 때 인덱스가 있으면 전부 불러오는게 아니라 해당 row만 불러옴
